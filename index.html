The Challenge in Cybersecurity

  • Cyberattacks are preventable with shared intelligence, but IoCs (e.g., malicious IPs, URLs) remain isolated.
  • Key Barriers:
    • Lack of trust between organizations.
    • Data privacy and reputation concerns.
    • Risk of tampering or misuse of shared data.
  • Result: Fragmented global defense against evolving threats.
Isolated Silos → Broken Chain

Proposed Solution

  • A decentralized blockchain-based platform for crowdsourced cyber threat intelligence.
  • Key Features:
    • Anonymous sharing of IoCs (IPs, hashes, domains, etc.).
    • Verify authenticity and integrity via smart contracts.
    • Trustless environment—no central authority needed.
Users → Platform → Blockchain

System Architecture: Blockchain Layer

  • Core: Smart contracts on Ethereum (or similar) for immutable storage.
  • Functions:
    • Store IoCs permanently (no edits/deletes).
    • Prevent duplicates and track submissions (anonymous via addresses).
    • Enable public retrieval and verification.

System Architecture: Backend & Frontend

Backend (Node.js + Express.js)

  • User authentication (wallets/keys).
  • Blockchain interaction via Web3.js/Ethers.js.
  • Validate and submit IoCs.

Frontend (React.js or HTML/JS)

  • Intuitive UI for submitting, browsing, and searching threats.
  • Verify via transaction IDs.
  • Promotes community-driven sharing.
Frontend → Backend → Blockchain

Ensuring Security and Privacy

  • Unique blockchain addresses (no real identities).
  • Digital signatures for IoC integrity.
  • Publicly verifiable but source-anonymous data.
  • Smart contracts automate tamper-proof logic.

Step-by-Step Workflow

  1. 1. Analyst discovers IoC (e.g., malicious IP).
  2. 2. Logs in via blockchain wallet.
  3. 3. Submits via frontend form.
  4. 4. Backend validates and sends to smart contract.
  5. 5. Blockchain records IoC immutably.
  6. 6. Others view/verify the data.
User Input → Validation → Immutable Storage → Global Access

Real-World Application

  • Scenario: Firm detects phishing domain.
  • Action: Upload IoC (domain + details) anonymously.
  • Blockchain: Stores immutably via smart contract.
  • Outcome: Other orgs block it proactively, preventing attacks.
  • Transforms siloed efforts into a collaborative ecosystem.
Phishing Detected → Shared → Blocked Globally

Innovation and Value

  • Trustless sharing without central servers.
  • Blockchain for guaranteed authenticity via consensus.
  • Anonymized global collaboration.
  • Broader Impact: Unified cybersecurity defense; potential for research publications.
New Era: Trustless + Collaborative Security
